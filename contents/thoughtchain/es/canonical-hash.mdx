## La huella digital inmutable de la ficción

Dentro de la Lit3, el **canon narrativo** no es solo un acuerdo cultural compartido; es una **garantía criptográfica**.

El marco de registro (Ledger Framework), como se introdujo anteriormente, establece el registro oficial de un artefacto literario. La clave de su autoridad es solamente un pequeño fragmento de datos: el **hash canónico**. Este hash es la huella digital de una narrativa, proporcionando un anclaje inmutable para cada obra Lit3, desde un solo poema hasta el capítulo de una novela completa.

Este artículo detalla el proceso técnico y las implicaciones narrativas de usar el hash canónico para lograr la **integridad literaria perpetua** en la blockchain.

---

## Propósito: Resolviendo el problema de la integridad digital

En la era de la Web2, un texto digital no tiene integridad inherente. El archivo original puede ser copiado, alterado y vuelto a subir infinitamente sin detección. El lector siempre debe confiar en la plataforma (Amazon, el sitio web de un editor, etc.) para proporcionar el texto auténtico.

El hash canónico invierte esta dinámica. Empodera al lector para verificar la autenticidad del texto, eliminando la necesidad de confiar en un proveedor centralizado.

| | Método Tradicional | Método Hash Canónico Lit3 |
| :--- | :--- | :--- |
| **Fuente de Integridad** | Confianza en el editor/plataforma. | **Prueba criptográfica** almacenada on-chain. |
| **Verificación** | Verificar la marca/URL de la fuente. | **Calcular el hash** y comparar con el registro. |
| **Vulnerabilidad** | Hackeo del servidor, corrupción de archivos o edición maliciosa. | **Registro inmutable** que no puede ser alterado. |

El hash canónico es la respuesta definitiva a la pregunta: **"¿Es este texto el que fue canonizado por el creador?"**

## Implementación técnica: La función unidireccional

El hash canónico se genera usando una **función hash criptográfica**, específicamente **SHA-256**, que se implementa en la Máquina Virtual Ethereum (EVM) como un contrato precompilado.

### 1. El proceso de hashing

Una función hash actúa como una licuadora determinista y unidireccional con tres propiedades clave:

* **Salida fija:** No importa el tamaño de la entrada — ya sea un haiku o un manuscrito de 500 páginas — la salida es siempre una cadena de longitud fija de 32 bytes (`bytes32`).
* **Irreversible:** No puedes revertir el hash para recuperar el texto original. Es una calle de un solo sentido, lo cual es esencial para la seguridad.
* **Efecto avalancha:** El más mínimo cambio en la entrada (por ejemplo, añadir una sola coma o un espacio) resulta en una salida hash completamente irreconocible y diferente.

### 2. La integración con el registro

Para asegurar el texto canónico, el proceso es:

1.  **Preparación (off-chain):** El autor Lit3 o la plataforma de publicación toma el archivo de texto final. Se ejecuta el algoritmo **SHA-256** sobre todo el contenido para generar el hash de 32 bytes.

2.  **Archivo (on-chain):** Este hash resultante se registra permanentemente en el almacenamiento del contrato inteligente del registro Lit3:

```solidity
struct Entry {
    // ... (otros campos de metadatos)
    bytes32 contentHash; // La huella digital inmutable
}
```

3.  **La Prueba:** Una vez que la transacción se confirma en cadena, el `contentHash` queda fijado para siempre. Este valor de 32 bytes es ahora la **representación matemática del texto canónico.**

## Implicaciones: Construyendo confianza narrativa

La integración del Hash Canónico es el **cambio de paradigma** para la Lit3 porque mueve la autoridad de la historia de una institución falible a una prueba matemática incorruptible.

### 1. Integridad perpetua y archivo

* **Preservación descentralizada:** El texto en sí puede almacenarse off-chain, pero la **prueba de su autenticidad** vive para siempre en la blockchain. Incluso si el enlace off-chain se rompe, mientras exista una sola copia del texto original en cualquier lugar, su autenticidad siempre puede verificarse contra el hash on-chain. Esta es la **solución de archivo definitiva** para la literatura.

### 2. Eliminando la disputa del canon

* **No más debates:** La cuestión de qué constituye la versión "original" o "canónica" se elimina del ámbito del debate literario y se convierte en un hecho verificable. Si dos versiones de un texto están circulando, un lector simplemente puede hashear ambas.
    * Si el hash de una versión **coincide** con el `canonicalHash` en el registro, es el artículo genuino.
    * Si **no coincide**, es una copia alterada, una edición de fans o un borrador no canónico.

### 3. Habilitando la propiedad verificable

* **Utilidad del NFT:** Un NFT que representa una obra Lit3 gana valor intrínseco más allá de su naturaleza visual o coleccionable. El tenedor del token no solo posee un archivo digital; posee los derechos a una **edición canónica certificada criptográficamente**. El hash on-chain es la prueba verificable de que la edición es la pieza genuina e inalterada del canon narrativo.
